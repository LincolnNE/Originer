# 🧠 UX Flow & IA

## 오리지너 – AI 강사 (MVP v1.0)

---

## 0. UX 설계 원칙 (매우 중요)

1. **“AI 같다”는 느낌 제거**
2. 질문 → 답변이 아니라 **수업 흐름**
3. 사용자는 **고민 안 하고 질문**해야 함
4. 강사는 **통제권을 잃지 않음**

> UX 키워드:
> 
> 
> **사람 같음 / 맥락 기억 / 부담 없음 / 통제 가능**
> 

---

## 1️⃣ 전체 정보 구조 (IA)

```
[Landing]
 ├─ 강사 선택
 ├─ AI 강사 소개
 └─ 수업 시작

[Student Area]
 ├─ 수업(Chat)
 │   ├─ 개념 설명
 │   ├─ 질문
 │   ├─ 재설명
 │   └─ 실습 피드백
 └─ 학습 기록 (읽기 전용)

[Instructor Area]
 ├─ AI 강사 생성
 ├─ 강의 데이터 관리
 ├─ AI 응답 미리보기
 └─ 대시보드

```

---

## 2️⃣ 학생 UX 플로우 (핵심)

### 📍 2-1. 강사 선택 화면

**목적**

- “이 AI는 누구인가?”를 즉시 인식

**구성**

- 강사 사진 / 이름
- 한 줄 설명
    
    > “이 강사는 개념을 비유로 잘 설명합니다”
    > 
- 강의 범위 태그
    
    (`Python`, `AI`, `LLM`, `초급`)
    

**CTA**

- ▶️ `이 강사에게 배우기`

📌 UX 포인트

→ **AI라는 단어 최소화**

→ “OOO 강사에게 질문하기” 톤 사용

---

### 📍 2-2. 수업 시작 (Chat Teaching View)

> 이 화면이 서비스의 심장
> 

### 화면 구성 (좌 → 우)

```
[강의 흐름 패널] | [대화 영역]

```

---

### 🔹 좌측: 강의 흐름 패널 (Context Panel)

- 현재 단원
- 진행 중인 개념
- 이전에 배운 항목 ✔️
- “지금 배우는 이유” 한 줄

📌 숨겨진 효과

→ 학생이 **길 잃었다는 느낌을 안 받음**

---

### 🔹 우측: 대화 영역 (Main Chat)

### AI 첫 멘트 예시

> “지금은 Transformer 구조를 설명할 차례야.
> 
> 
> 먼저 전체 그림부터 볼까?”
> 

### 입력 UX

- placeholder:
    
    > “모르는 걸 그냥 적어도 돼요”
    > 

### 빠른 액션 버튼

- `다시 설명`
- `예시로`
- `코드로 보여줘`
- `한 단계만 더`

📌 중요

→ **학생이 ‘어떻게 질문할지’ 고민하지 않게**

---

### 📍 2-3. 재설명 UX (차별 포인트)

학생:

> “이 부분 잘 모르겠어요”
> 

AI 강사 반응:

1. ❓ 이해도 체크
    
    > “어디까지는 이해됐어?”
    > 
2. 🎚 설명 깊이 조절
3. 🧩 다른 비유 사용

📌 UX 규칙

- 같은 설명 반복 ❌
- **표현 방식만 바꿔 재설명 ⭕**

---

### 📍 2-4. 실습 피드백 UX

학생 코드 입력 시:

AI 응답 구조:

1. 👍 잘한 점
2. ⚠️ 문제 지점
3. 🤔 질문 유도
    
    > “이 부분에서 왜 이렇게 생각했어?”
    > 

📌 절대 금지

- 바로 정답 코드 제공 ❌

---

## 3️⃣ 학생 학습 기록 UX (Passive)

**목적**

- 복습용
- 성취감 제공

**구성**

- 배운 개념 타임라인
- 헷갈린 개념 표시
- 재설명 횟수

📌 UX 톤

→ 평가 ❌

→ **기록 & 성장** ⭕

---

## 4️⃣ 강사 UX 플로우

---

### 📍 4-1. AI 강사 생성 화면

**단계형 UX (Step UI)**

1️⃣ 강사 정보 입력

2️⃣ 강의 자료 업로드

3️⃣ AI 학습 중 상태 표시

4️⃣ 미리보기 테스트

📌 중요

→ “학습 중입니다…”가 아니라

→ **“강의 스타일을 분석 중입니다”**

---

### 📍 4-2. AI 응답 미리보기

강사는 다음을 확인 가능:

- 질문 입력
- AI 답변 확인
- ❌ / ⭕ 체크
- 수정 코멘트

📌 UX 포인트

→ 강사가 **AI를 훈련한다는 느낌**

---

### 📍 4-3. 강사 대시보드

### 주요 카드

- 질문 TOP 5
- 재설명 많은 구간
- 학생 이탈 구간

### 인사이트 문구 예시

> “이 단원에서 학생들이 가장 많이 막혔어요”
> 

---

## 5️⃣ 톤 & 카피 가이드 (중요)

### ❌ 피해야 할 톤

- “AI가 분석했습니다”
- “모델이 판단한 결과”

### ⭕ 권장 톤

- “내가 이렇게 설명해볼게”
- “아까 말한 것 기억나?”

📌 **항상 ‘강사’가 말하는 것처럼**

---

## 6️⃣ MVP UX 성공 기준

- 학생이 **AI라고 인식하지 않고 질문**
- “다시 설명해줘”가 자연스럽게 눌림
- 강사가 **AI 답변을 신뢰**
