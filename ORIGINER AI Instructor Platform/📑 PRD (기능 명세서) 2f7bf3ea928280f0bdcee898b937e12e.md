# 📑 PRD (기능 명세서)

## **오리지너 – AI 강사 (ORIGINER: AI Instructor Platform)**

---

## 0. 문서 개요

- **문서 목적**: 오리지너 MVP 개발을 위한 기능·요건 정의
- **대상**: Product / Engineering / Design / AI 팀
- **범위**: MVP v1.0
- **비범위**: 음성 강의, 마켓플레이스, 멀티 강사 비교

---

## 1. 제품 목표 (Product Goals)

### 🎯 핵심 목표

1. **강사 1명을 AI 강사로 복제**
2. **학생이 실제 강사처럼 느끼는 학습 경험 제공**
3. **강사의 질문 대응 부담 80% 이상 감소**

### ❌ 하지 않는 것 (Out of Scope)

- 범용 AI 챗봇
- 자동 강의 영상 생성
- 시험 감독/평가 시스템

---

## 2. 사용자 정의 (User Personas)

### 👨‍🏫 강사 (Instructor)

- AI / 프로그래밍 강의 경험자
- 반복 질문에 피로
- 자신의 강의 스타일을 유지하고 싶음

### 👩‍🎓 학생 (Learner)

- 비전공자 또는 주니어
- 질문 타이밍이 잦음
- “다시 설명”을 부담 없이 원함

---

## 3. 사용자 시나리오 (Core User Flow)

### 3.1 강사 플로우

1. 강사 가입
2. 강의 자료 업로드
3. AI 강사 생성 요청
4. 미리보기 테스트
5. 학생에게 공유

### 3.2 학생 플로우

1. 강사 선택
2. 수업 시작
3. 질문 / 재질문
4. 실습 피드백
5. 학습 이력 누적

---

## 4. 핵심 기능 명세 (Functional Requirements)

---

### 🧩 F-1. 강사 오리지닝 (Instructor Origination)

**설명**

강사의 강의 스타일을 학습하여 AI 강사를 생성

**입력**

- PDF / PPT
- 텍스트 강의 노트
- 코드 예제
- Q&A 로그 (선택)

**출력**

- Instructor Profile
- Explanation Style Vector
- Difficulty Scaling Rules

**요건**

- 업로드 파일 자동 분류
- 강의 단원 구조 추출
- 설명 패턴 저장

**성공 기준**

- 동일 질문 시 일관된 답변 스타일 유지

---

### 🎓 F-2. AI 강사 수업 (Chat-based Teaching)

**설명**

학생과 대화형 수업 제공

**기능**

- 개념 설명
- 단계별 재설명
- 예시 코드 제공
- 질문 히스토리 기억

**제약**

- 강사 스타일 벗어나는 답변 금지
- “모르면 모른다고 말하기”

**UX 포인트**

- “이해됐어?” 같은 자연스러운 체크

---

### 🧠 F-3. 학생 메모리 시스템 (Learner Memory)

**설명**

학생의 학습 상태를 누적 관리

**저장 항목**

- 이해한 개념
- 헷갈린 포인트
- 질문 빈도
- 재설명 횟수

**활용**

- 설명 난이도 자동 조절
- 반복 설명 최소화

---

### 🛠 F-4. 실습 & 과제 피드백 (MVP-lite)

**설명**

학생이 입력한 코드/답변에 대해 피드백

**범위**

- 코드 오류 지적
- 접근 방식 피드백
- 정답 제공 ❌

---

### 📊 F-5. 강사 대시보드 (Instructor Dashboard)

**지표**

- 질문 TOP 10
- 학생 이탈 지점
- 재설명 발생 구간

**기능**

- AI 답변 로그 확인
- 강의 개선 추천

---

## 5. 비기능 요구사항 (Non-Functional)

### ⚡ 성능

- 응답 시간 < 2초 (P95)

### 🔐 보안

- 강사 데이터 독립 저장
- 강사별 벡터 완전 분리

### 📈 확장성

- 강사 수 증가 시 성능 저하 없음
- 학생 수 수평 확장

---

## 6. 기술 요구사항 (Technical Requirements)

### AI

- LLM + RAG 구조
- Instructor Vector Store
- Learner Memory Store 분리

### Backend

- REST + Streaming
- Session-based Context 관리

### Frontend

- Web Chat UI
- 코드 블록 하이라이트
- 단계 표시 UI

---

## 7. KPI & 성공 지표

| 지표 | 목표 |
| --- | --- |
| 질문 응답 정확도 | 85%+ |
| 재질문 감소율 | 30%+ |
| 강사 질문 부담 감소 | 80% |
| 학습 지속률 | 기존 대비 2x |
